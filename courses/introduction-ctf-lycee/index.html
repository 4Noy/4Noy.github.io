Crypto/XOR Properties

Il y a quatre principales propriétés que nous devrions considérer lorsque nous résolvons des défis en utilisant l'opérateur XOR :

Commutative : A ⊕ B = B ⊕ A
Associative : A ⊕ (B ⊕ C) = (A ⊕ B) ⊕ C
Identité : A ⊕ 0 = A
Auto-inverse : A ⊕ A = 0

Décortiquons cela. Commutative signifie que l'ordre des opérations XOR n'est pas important. Associative signifie qu'une chaîne d'opérations peut être effectuée sans ordre (nous n'avons pas besoin de nous soucier des parenthèses). L'identité est 0, donc XOR avec 0 "ne fait rien", et enfin quelque chose XOR'd avec lui-même retourne zéro. Mettez cela en pratique !
